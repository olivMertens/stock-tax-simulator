Plan technique Construire un produit logiciel de bout en bout

1 Stack moderne

React 19TypeScript 6Vite 8Tailwind 4

Lazy-loading, memoization, custom hooks, Error Boundary — patterns de production, pas un prototype.

2 Architecture modulaire

Moteur fiscal séparé de l'UI en 6 modules indépendants. Logique métier testable sans composants graphiques.

3 Tests automatisés

VitestTesting Library

9 fichiers de tests : moteur fiscal, parser CSV, déclaration, stockage, taux ECB, parser avis d'imposition, composants.

4 Intégration d'APIs

ECBFinnhubAzure Function

Taux EUR/USD historiques (ECB), cours MSFT live (Finnhub via proxy serverless). Cache intelligent côté client et serveur.

5 Sécurité web

CSPX-Frame-OptionsPermissions-Policy

Headers OWASP, clé API protégée côté serveur, worker PDF local (pas de CDN tiers).

6 PWA & offline

Service Worker, manifest, cache-first pour les assets. L'app est installable sur mobile.

7 Parsing avancé

PDF (pdfjs)CSV (PapaParse)

Import CSV Fidelity avec réassemblage des séparateurs de milliers. Extraction d'avis d'imposition PDF via coordonnées XY.

8 Persistance versionnée

Schéma de données versionné dans localStorage avec migration automatique et validation/sanitisation des entrées.

Plan fonctionnel / fiscal Maîtriser un domaine métier complexe sans être fiscaliste

1 4 régimes de stock awards

ESPPStock AwardAGA MacronAGA pré-Macron

Chaque type a un traitement fiscal distinct (gain d'acquisition, abattements, PS activité vs patrimoine).

2 Double imposition

Séparation gain d'acquisition / plus-value de cession. Imputation croisée des moins-values sur le gain d'acquisition.

3 PFU vs barème progressif

Comparateur automatique avec parts fiscales, quotient familial, CSG déductible, abattement durée de détention, CEHR.

4 Barèmes multi-exercices

202420252026

Tranches IR, hausse CSG 2025 (PS patrimoine 17.2% → 18.6%), seuil AGA 300k€, CEHR, plafond QF.

5 Aide à la déclaration

Génération automatique des cases 2042, 2042-C et formulaire 2074 (3VG, 3VH, 1TZ, 1WZ, 1TT, 2OP, 3SG…).

6 Conversion EUR/USD réglementaire

Taux BCE au jour de l'acquisition pour chaque lot. Gestion week-ends/jours fériés avec look-back 7 jours.

Plan opérationnel Mener un projet produit en autonomie

1 Identification du pain point

Complexité fiscale réelle des employés MSFT France (4 plans, double imposition, conversion USD). Aucun outil interne existant.

2 Product thinking

Flux complet

Import Fidelity → Import avis d'impôt → Simulation temps réel → Comparaison régimes → Instructions déclaration. End-to-end.

3 Déploiement Azure

Static Web AppsAzure Functions

Utilisation de l'écosystème Microsoft de bout en bout. CI/CD intégré.

4 IA comme levier

GitHub Copilot

En tant que non-dev, piloter la création d'une app de 3 000+ lignes testées et déployées. Cas d'usage concret de l'IA comme multiplicateur de capacité.
